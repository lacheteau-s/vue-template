# vue-template
A boilerplate for [Vue](https://vuejs.org/) projects using Vue 3 with [TypeScript](https://www.typescriptlang.org/), [Single-File Components](https://vuejs.org/guide/scaling-up/sfc), and [Composition API](https://vuejs.org/guide/extras/composition-api-faq.html).

This template is built on top of the code generated by [vuejs/create-vue](https://github.com/vuejs/create-vue) (the official Vue project scaffolding tool) with the following configuration:

```
✔️ TypeScript
❌ JSX Support
❌ Vue Router
❌ Pinia
❌ Vitest
❌ End-to-End Testing Solution
❌ ESLint
```

With the exception of TypeScript, any other feature implemented in this template (e.g. [ESLint](https://eslint.org/)) was added and configured manually for my own personal [motive](#motive).

While `vuejs/create-vue` creates a generic template with a bunch of features pre-configured to fit most use cases, this template is an opinionated, stripped-down version.

As such, it is **NOT** meant as an alternative to `vuejs/create-vue`.
For all intents and purposes, I do recommend that you use `vuejs/create-vue` to initialize your Vue projects.

## How to use

Simply copy the `template` folder to the desired location, and follow the template's [README.md](template/README.md) (don't forget to edit the title and description).

### Clone and copy

```sh
git clone 
cd vue-template
cp -R template <target-location>
```

While this method is the most straightforward, it forces you clone the repository first.

### degit

```
npm install -g degit
degit github:lacheteau-s/vue-template/template[#<branch>] <outdir>
```

| Option | Default | Description |
|--------|---------|-------------|
| `#<branch>` | `template/base` | Retrieves a specific version of this template with [additional features](#additional-features) |
| `<outdir>` | `template` | Specifies the output directory |

Example:

```
degit github:lacheteau-s/vue-template/template#template/vuetify MyApp
```

This will fetch the template with `Vuetify` included and put it in a directory named `MyApp`.

For more information, see [degit](https://github.com/Rich-Harris/degit).

## Additional features

Several additional features are available in different branches, all derived from the base template. Each version is tagged adequately:

| Tag | Features |
|-----|----------|
| `base` (current) | Vue + TypeScript + ESLint + Prettier |
| `vuetify` | `base` + Vuetify |
| `router` | `base` + Vue Router |
| `pinia` | `base` + `pinia` |
| `router-pinia` | `base` + `router` + `pinia` |
| `vuetify-router` | `base` + `vuetify` + `router` |
| `vuetify-pinia` | `base` + `vuetify` + `pinia` |
| `full` | `base` + `vuetify` + `router` + `pinia` |

For more information about each template, refer to this README.md from the appropriate version.

### Why not implement an npm initializer?

After a brief consideration, I determined that it wouldn't be worth the effort for something that is meant for my own educational purpose (see below), and therefore not meant to be published.

## Motive

This template is part of a personal initiative to catch up with web development after 10 years of focusing mostly on desktop, mobile, and backend development.

I chose Vue because I found it more appealing than Angular and React.
It seems to have been [inspired](https://012.vuejs.org/guide/) by the [MVVM](https://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93viewmodel) pattern which I have had quite a lot of positive experience with, working with [WPF](https://learn.microsoft.com/en-us/dotnet/desktop/wpf/overview), and [Xamarin/.NET MAUI](https://dotnet.microsoft.com/en-us/apps/maui).

My end goal here wasn't to merely learn Vue but to also understand the whole ecosystem around it (like [Vite](https://vite.dev/)).
So instead of just initializing a new project with `vuejs/create-vue`, I dug into every file in an effort to understand why things are the way they are, and did a few tweaks where I deemed appropriate, all while gaining precious knowledge.
